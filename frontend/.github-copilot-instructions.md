# GitHub Copilot Instructions - Frontend

## Project Overview

- **Name:** "Toomas633's Dungeon" - Personal projects homepage frontend (v4.2.1)
- **Stack:** Vue 3 + TypeScript + Vuetify 3 + Vite
- **Purpose:** Personal portfolio/homepage showcasing projects, demos, and providing contact functionality
- **Build Tool:** Vite with extensive plugin ecosystem for optimization
- **UI Framework:** Vuetify 3 (Material Design components)
- **Routing:** Vue Router 4 with modular route organization
- **State Management:** Composables and services (no Vuex/Pinia)

## Architecture & Structure

### Core Technologies

- **Vue 3:** Composition API preferred, SFCs with `<script setup>`
- **TypeScript:** Strict typing throughout, custom types in `src/types/`
- **Vuetify 3:** Material Design UI components, custom theming
- **Vite:** Modern build tool with HMR, plugin-based optimization
- **Vue Router 4:** File-based modular routing system

### Directory Structure

```
src/
├── assets/           # Static assets (SCSS, images, icons, JSON data)
├── components/       # Reusable Vue components
│   └── cookies/     # Cookie consent related components
├── constants/        # App constants and environment config
├── enums/           # TypeScript enums
├── helpers/         # Utility functions and composables
├── plugins/         # Vue plugins (Vuetify config)
├── router/          # Modular routing (main, projects, demos, archive, servers)
├── services/        # API services (email, GitHub, Minecraft)
├── types/           # TypeScript type definitions
├── util/            # Utilities (event bus, etc.)
└── views/           # Page components organized by section
```

## Development Patterns

### Component Structure

- Use `<script setup>` syntax with TypeScript
- Props with proper TypeScript interfaces
- Emit events with typed signatures
- Composables for shared logic
- Vuetify components for UI consistency

```vue
<script setup lang="ts">
interface Props {
	title: string
	items: MenuItem[]
}

interface Emits {
	(e: 'select', item: MenuItem): void
}

const props = defineProps<Props>()
const emit = defineEmits<Emits>()
</script>
```

### Routing Patterns

- Modular routing in separate files (`router/`)

## Common Tasks

### Adding New Components

1. Create `.vue` file in appropriate `src/components/` subfolder
2. Use TypeScript and proper prop/emit typing
3. Follow Vuetify design patterns
4. Export from parent directory if needed

### Adding New Routes

1. Add route definition to appropriate `src/router/` file
2. Create corresponding view component in `src/views/`
3. Update route metadata for SEO
4. Test navigation and breadcrumbs

### Adding New Services

1. Create service file in `src/services/`
2. Define TypeScript interfaces in `src/types/`
3. Implement error handling and response typing
4. Add environment configuration if needed

### Updating Dependencies

1. Check compatibility with Vue 3 and Vuetify 3
2. Update TypeScript types if needed
3. Test build process and development server
4. Update documentation if APIs change

Remember: This is a modern Vue 3 application emphasizing type safety, performance, and maintainability. Always consider the user experience and follow Vue 3 best practices.
